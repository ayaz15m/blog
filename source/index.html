Week 4

A quarter of the way done! We started getting into the good stuff, RAILS! It was awesome to see how ruby, HTML, and CSS all come together in rails. We began by going over generating a new rails app, model, and a saffold. We also went over model relationships such as: beloongs to, has many, and many to many.

This week the the project consisted of making a rails app to track different homework assignments for different users. The users are able to login and see their assignments divided respectively into past due, not done, and completed tables. Past due assignments are also highlighted. The user is also able to complete the homework assignment and add the assignment url. For the done table, comments can be added for each assignments.

This project was more difficult compared to the other projects. I had a hard time starting out trying to figure out generating the models and scaffolds. I ended using one scaffold model which seemed to work out nicely. The consequence of doing this was that seeding the model was not as concise if there would have been multiple models.


Week 3

The material covered in week 3 was a little bit different as we began to steer away from ruby and gotinto TDD, Git, Middleman, HTML. I have never done testing before so the concept of incorporating into the coding process was not that clear for me. But I think like with any concept, practice and exposure over time will get me more accustomed to it.

This week our weekend project consisted of writing a program with some kind of character which can be tested and use git to put it onto Github. Then we needed to create a HTML describing the project. For my program I made a bank account which allows the user to withdraw and make deposits. The tests I did for this project included: the balance is given without doing anything, the balance after a deposit is made, the balance after a withdrawal is made, and the balance after a deposit and a withdrawal are made together.

After adding the rest of the code, which displays output and lets user input info, I had some trouble running the tests afterwards because I was trying to use different values. This problem got solved by have the code required for the tests in one class and the rest of the code in another class. Once the 2 classes were made everything worked smoothly!
